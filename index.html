Функціонал
Що буде зроблено у проекті
👤
Авторизація та профіль
Реєстрація та вхід за допомогою JWT токенів. Профіль зберігає ціль дієти, вагу, зріст, вік та активність. Автоматичний розрахунок TDEE (добова норма калорій) за формулою Mifflin-St Jeor.
MVP JWT Спринт 2
🥗
Каталог рецептів
База 100–150 рецептів з категоріями (сушка, кето, набір маси, підтримання). Фільтри за категорією, часом готування та виключеними продуктами. Детальна сторінка з БЖУ та інструкцією.
MVP Фільтри Спринт 2–3
📊
Трекінг харчування
Щоденник їжі — додавання страв із рецептів або вручну. Автоматичний підрахунок калорій та БЖУ за день. Порівняння з нормою та прогрес-бари у реальному часі.
MVP Charts Спринт 3
🛒
Список покупок
Автоматична генерація списку інгредієнтів із вибраних рецептів. Підсумування однакових продуктів, чекліст із відмітками, можливість редагування та очищення.
MVP Спринт 3
📈
Аналітика та графіки
Дашборд із графіком динаміки ваги та споживання калорій за тиждень/місяць. Кругова діаграма розподілу БЖУ. Рекомендації по коригуванню раціону. Реалізовано через Chart.js.
Chart.js Розширення Спринт 4
🔒
Безпека та логування
Сервер логує всі HTTP-запити у JSON-файл (метод, URL, тіло, статус, час). Валідація всіх вхідних даних на сервері (тонкий клієнт). Захист маршрутів через JWT middleware.
MVP Логування Спринт 2
Roadmap / Спринти
Поетапний план розробки
1
Інфраструктура та налаштування проекту
Репозиторій, структура папок, підключення БД, базовий сервер
Тиждень 1
▼

Backend задачі

  • Ініціалізація Node.js + Express проекту (npm init)
  • Підключення MongoDB Atlas + налаштування Mongoose
  • Базова структура папок: routes/, controllers/, models/, middleware/
  • Налаштування CORS, JSON body parser, .env файлу
  • Реалізація модуля логування у JSON-файл (logger.js)

Frontend та DevOps задачі

  • Ініціалізація React додатку (Create React App / Vite)
  • Налаштування Axios для HTTP-запитів до API
  • Реєстрація командного репозиторію на GitHub
  • Створення Trello дошки, колонки Kanban
  • Надання доступу до GitHub + Trello викладачу
2
Авторизація + моделі даних + базовий API
JWT-авторизація, схеми MongoDB, CRUD для рецептів
Тижні 2–3
▼

Backend задачі

  • Mongoose Schema: User (email, password, profile, goal)
  • Mongoose Schema: Recipe (name, category, calories, BJU, ingredients)
  • POST /api/auth/register — реєстрація + bcrypt хешування пароля
  • POST /api/auth/login — вхід + генерація JWT токена
  • JWT middleware для захисту приватних маршрутів
  • GET/POST /api/recipes — список та додавання рецептів
  • Seed-скрипт: заповнення 100+ рецептів у MongoDB

Frontend задачі

  • Сторінка реєстрації та входу (форми, валідація)
  • Збереження JWT токена у пам'яті (context/state)
  • Protected Routes — redirect якщо не авторизований
  • Онбординг: вибір цілі дієти після реєстрації
  • Сторінка каталогу рецептів — базовий список карток
3
Основний функціонал: рецепти + трекінг + покупки
Фільтри рецептів, щоденник їжі, список покупок, розрахунок калорій
Тижні 4–5
▼

Backend задачі

  • Mongoose Schema: DailyLog (userId, date, meals[], totalCalories)
  • POST /api/logs — додати запис у щоденник
  • GET /api/logs?date=2026-04-05 — отримати записи за день
  • GET /api/recipes?category=кето&exclude=milk — фільтрація
  • Розрахунок TDEE (Mifflin-St Jeor) у profileController

Frontend задачі

  • Фільтри на сторінці рецептів (категорія, виключення)
  • Детальна сторінка рецепту + кнопка "Додати в день"
  • Сторінка щоденника — список страв + підрахунок БЖУ
  • Прогрес-бари (спожито / норма) у реальному часі
  • Сторінка списку покупок — генерація + чекліст
4
Аналітика, дашборд та тестування
Графіки Chart.js, дашборд, тест-документація, виправлення багів
Тижні 6–7
▼

Аналітика та Backend

  • GET /api/logs/stats?period=week — статистика за тиждень
  • Агрегація калорій по днях через MongoDB pipeline
  • Лінійний графік калорій за тиждень (Chart.js)
  • Кругова діаграма розподілу БЖУ за день
  • Дашборд: KPI-картки + графіки + останні рецепти

Тестування (QA)

  • Тест-кейси для всіх API-маршрутів (Thunder Client)
  • Функціональне тестування форм авторизації
  • Тестування фільтрів рецептів (позитивні + негативні)
  • Тестування трекінгу: додавання / видалення їжі
  • Формування звіту по тестуванню у Google Docs
